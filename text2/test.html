使用 git clone 本仓库及所有 submodules，然后设置部署所需环境变量

这里使用 git clone 命令来克隆一个 Git 仓库时，如果该仓库包含了子模块（submodules），需要通过加上 –recursive 选项来克隆本仓库及其所有子模块。
这样不会导致下载的包中相关的子模块都是空的的情况
若克隆太慢可以通过设置git参数解决

8. 问题

  1. 在window中直接从github上下载压缩包源码，会发现eggroll、fateboard和fateflow三个文件夹为空

解决：需要分别下载对应的压缩包，解压后放到对应的文件夹中

  1. 在解压fateboard的过程中遇到一些问题
    ！解压fateboard
    解压 Linux 内核报错 ( Can not create symbolic link : 客户端没有所需的特权 )

解决：右键7zFM.exe，选择“以管理员身份运行”即可

  1. 使用Ubuntu22.04，自带python版本为3.10会出现一系列问题（集中出现在安装依赖部分）

解决：安装python3.8，然后再安装python3.8-dev，再执行安装依赖的命令(注意要用python3.8替换python3执行)

  1. 启动fateboard这一步：ln -s target/fateboard-1.11.2.jar fateboard.jar报错
    解决：注意检查文件fateboard-1.11.2.jar的实际版本号
